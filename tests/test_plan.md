# Checklist breve de validación

## 1) Checklist de pruebas mínimas
- [ ] Ejecuté al menos 2 casos válidos.
- [ ] Ejecuté al menos 1 caso inválido.
- [ ] Registré resultados esperados y obtenidos.

## 2) Checklist de documentación
- [ ] Llené `docs/propuesta.md` completamente.
- [ ] Llené `docs/caso_de_uso.md` con flujo principal y alternativo.
- [ ] Llené `docs/plan_de_pruebas.md` con tabla y estado.
- [ ] Revisé `docs/estructura_repositorio.md` y respeté la organización.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` se ejecuta sin errores críticos.
- [ ] Existe `src/main.<ext>` de acuerdo con mi lenguaje.
- [ ] El prototipo muestra salida entendible en terminal.

## 4) Checklist de entrega final
- [ ] El proyecto se mantiene pequeño y viable.
- [ ] No usé frameworks ni servicios externos.
- [ ] El repositorio está limpio, ordenado y con nombres claros.
- [ ] Subí todos los archivos obligatorios al repositorio.
